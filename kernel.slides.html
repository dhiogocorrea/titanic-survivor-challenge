Titanic Survivor Challenge

Esse desafio visa a criao de um modelo de classificao para a predio de sobreviventes do navio Titanic analisando algumas variveis:

  1. PassangerId: Id nico do passageiro
  2. PClass: Classe que o passageiro viajou (1, 2 ou 3)
  3. Name: Nome do passageiro
  4. Age: Idade do passageiro
  5. SibSp: Nmero de irmos e cnjuges viajando junto do passageiro
  6. Parch: Nmero de pai/me e filhos viajando junto do passageiro
  7. Ticket: Nmero do ticket do passageiro
  8. Fare: Quantidade de tarifas que o passageiro pagou para embarcar
  9. Cabin: Nmero da cabine do passageio
  10. Embarked: Inicial do nome do porto que o passageiro embarcou (S, Q, C)
  11. Survived: 0 - No sobreviviu, 1 - Sobreviviu -> a varivel que queremos prever.

Duas bases esto disponveis: train.csv: Base de treinamento contendo a varivel Survived test.csv: Base que queremos prever (no contm Survived)

Esse notebook foi adaptado de vrias resolues j existentes para o problema no Kaggle (referncias no final).

Assunções baseadas na análise de dados¶

Chegamos aos seguintes pressupostos baseados na anlise de dados feita at agora. Podemos validar essas suposies antes de tomar as aes apropriadas.

Correlao.

Queremos saber quo bem cada recurso se correlaciona com "Survival".

Completude.

  1. Podemos querer completar a caracterstica "Age", pois possivelmente correlacionada com a sobrevivncia.
  2. Podemos querer completar a caracterstica "Embarked", pois ele tambm pode se correlacionar com a sobrevivncia ou outra caracterstica importante.

Correo.

  1. O recurso de ticket pode ser retirado da nossa anlise, pois contm uma alta taxa de duplicidade (22%) e pode no haver uma correlao entre o Ticket e a sobrevivncia.
  2. A caracterstica de "Cabin" pode ser descartado, pois altamente incompleto ou contm muitos valores nulos no treinamento e no conjunto de dados de teste.
  3. O PassengerId pode ser retirado do conjunto de dados de treinamento, pois no contribui para a sobrevivncia.
  4. O recurso de nome relativamente no-padro, pode no contribuir diretamente para a sobrevivncia, ento talvez seja descartado.

Criao.

  1. Podemos querer criar uma nova caracterstica chamado "Family" baseado no Parch e no SibSp para obter uma contagem total de membros da famlia a bordo.
  2. Podemos querer trabalhar com o Nome para extrair o ttulo como um novo recurso.
  3. Podemos querer criar uma nova caracterstica para ranges de idades. Isso transforma um recurso numrico contnuo em um recurso categrico ordinal.
  4. Tambm podemos querer criar um range de "Fare", se isso ajudar na nossa anlise.

Classificao.

Tambm podemos adicionar nossas suposies com base na descrio do problema mencionada anteriormente.

  1. Mulheres (sexo = mulheres) tinham maior probabilidade de ter sobrevivido.
  2. As crianas (idade <?) Eram mais propensas a ter sobrevivido.
  3. Os passageiros da classe alta (Pclass = 1) tinham maior probabilidade de ter sobrevivido.

                                  (dataset['Pclass'] == j+1)]['Age'].dropna()

            # age_mean = guess_df.mean()
            # age_std = guess_df.std()
            # age_guess = rnd.uniform(age_mean - age_std, age_mean + age_std)

            age_guess = guess_df.median()

            # Convert random age float to nearest .5 age
            guess_ages[i,j] = int( age_guess/0.5 + 0.5 ) * 0.5
            
    for i in range(0, 2):
        for j in range(0, 3):
            dataset.loc[ (dataset.Age.isnull()) & (dataset.Sex == i) & (dataset.Pclass == j+1),\
                    'Age'] = guess_ages[i,j]

    dataset['Age'] = dataset['Age'].astype(int)

Modelo, predição e resolução¶

Agora estamos prontos para treinar um modelo e prever a soluo necessria. Existem mais de 60 algoritmos de modelagem preditiva para escolher. Devemos entender o tipo de problema e o requisito de soluo para restringir a alguns modelos selecionados que podemos avaliar. Nosso problema um problema de classificao. Queremos identificar a relao entre sada (Sobreveu ou no) com outras variveis (Sexo, Idade, Embarque ...).

Estamos realizando uma categoria de aprendizado de mquina que chamada de aprendizado supervisionado, pois estamos treinando nosso modelo com um determinado conjunto de dados com classe conhecida. Com esses dois critrios - Aprendizado Supervisionado mais Classificao, podemos restringir nossa escolha de modelos a alguns. Esses incluem:

  • Support Vector Machines
  • Naive Bayes
  • rvore de deciso
